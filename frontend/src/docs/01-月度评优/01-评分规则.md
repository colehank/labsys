# 大组会评分规则

这份文档说明 **月度评优** 是怎么算出来的 —— 从三个数据来源，到五个维度，再到最终的综合名次。规则完全公开、可复核。

> 一句话版本：每个人在**报告、出勤、讨论**三方面的表现，被拆成**五个等权维度**，各自归一化后加权求和得到「组会分」；只有**全勤且讨论达标**的人进入评优候选，再由「组会名次」与「进展名次」双轨平均，排出当月之星。

## 一、三个数据来源

| 来源 | 怎么产生 | 对应维度 |
| --- | --- | --- |
| **报告评分** | 每次汇报后，大家在问卷星按 1–5 分打分 | 态度、精良、逻辑清晰 |
| **出勤** | 每次组会记录 到场 / 请假 / 缺席 | 出勤 |
| **讨论** | 每次组会记录的发言积分 | 讨论 |

报告评分只在**你作为汇报人**的那几次产生；出勤和讨论则**每次组会**都在记。工作坊、团建这类不评分的活动，同样会记考勤，只是不产生报告评分。

## 二、五个维度，各占 20%

综合分由五个维度**等权**合成，每个维度权重都是 **20%**：

1. **态度** —— 汇报的认真与投入程度
2. **精良** —— 材料与呈现的完成度
3. **逻辑清晰** —— 讲述的条理与严谨
4. **出勤** —— 评选期内的到场率
5. **讨论** —— 累计的发言积分

前三项来自报告评分（问卷星 1–5 分的平均），后两项来自出勤与讨论记录。

## 三、归一化：让不同维度可比

五个维度的原始量纲完全不同（分数、百分比、积分），不能直接相加。所以每个维度都先在**全体成员之间**做 **min-max 归一化**，映射到 `0 ~ 100`：

```
归一分 = (你的值 − 全体最小值) / (全体最大值 − 全体最小值) × 100
```

- 该维度最高的人得 `100`，最低的人得 `0`，其余按比例落在中间。
- 如果全体在某维度上完全相同（无区分度），则统一记 `50`。

这样五个维度就都在同一把尺子上，可以公平地加权。

## 四、组会分与组会名次

把五个归一分按权重加权平均，得到每个人的 **组会分（meeting）**：

```
组会分 = (态度ⁿ + 精良ⁿ + 逻辑ⁿ + 出勤ⁿ + 讨论ⁿ) × 20% / 权重和
```

按组会分从高到低排序，就得到 **组会名次**。

## 五、评优门槛（硬性资格）

综合分高不代表自动入选。要进入评优候选，必须同时满足两条**硬门槛**：

- **全勤** —— 评选期内出勤率需达 **100%**
- **讨论达标** —— 累计讨论积分 **≥ 4**

达不到门槛的人，即使报告分很高，也不进入当月候选名单。这两条鼓励「**持续到场 + 积极参与讨论**」，而不只是汇报当天表现好。

## 六、双轨合并：排出当月之星

进入候选后，最终名次由**两条轨道**平均决定：

- **组会名次** —— 上面算出的综合表现排名（在候选人之间重新计名次）
- **进展名次** —— 结合研究进展、由管理员维护的「进展顺序」排名

```
最终分 = (组会名次 + 进展名次) / 2      ← 数值越小越靠前
```

按最终分从小到大排序，就是当月评优结果。之所以取两轨平均，是为了**兼顾「这一个月的组会表现」与「研究本身的推进」**，避免任何单一维度决定全部。

## 七、常见问题

- **工作坊/团建也扣分吗？** 不评分，但记考勤；出勤照常计入出勤维度。
- **某次没被打分会怎样？** 只有你**作为汇报人**那几次才有报告评分；没轮到你汇报的组会不影响你的报告分平均。
- **规则会变吗？** 权重、门槛都是可配置项，若调整会提前公示，并在此文档更新。
